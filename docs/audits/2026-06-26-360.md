# 360-Degree Audit — 2026-06-26

Full code review, structural audit, and 360-degree analysis of the
`sensor_app` full-stack demo (Flask + Postgres backend, Angular frontend,
Docker Compose), performed with the methodology in
[solid-ai-templates](https://github.com/braboj/solid-ai-templates)
(`templates/base/workflow/360.md`, ADR-003).

Run as **seven parallel, context-isolated reviewers**, each starting with a
clean context and a single role prompt to prevent perspective bleed. Because
this app has a real user-facing surface, the three non-engineering lenses
(Value, Viability, Discovery) are kept, and the engineer "Quality" lens is
re-projected into four engineering dimensions (Backend code, Frontend code,
Architecture & Structure, Testing/CI) to satisfy the requested full code
review + structural audit. The overall grade is the lowest dimension grade.
Every crux finding was independently re-verified by the parent agent (git
ls-files, file existence, source reads) before grading.

> convention. The templates submodule was relocated from `docs/` to
> `docs/solid-ai-templates/` so that `docs/` is a real project folder (holding

## Current bottleneck

**Frontend & Testing (D)** — the product's two weakest slices fail for the
same root cause: *tutorial scaffolding was never cleaned up and the project
was never wired for real delivery.* The sole frontend test is triple-broken
**and** unrunnable (no `test` target in `angular.json`, no `test` script in
`package.json`); the API URL is hardcoded to `localhost:5000` with a dead
`REACT_APP_*` env var on an Angular app; the container ships `ng serve` as
"production"; and there is no CI at all despite the project's own roadmap
(`SOLUTION.md:35`, V0.0.6.0) promising GitHub Actions.

The single most *urgent* fix is the Critical RCE (`debug=True` on
`0.0.0.0` as the Docker `CMD`). The single highest-*leverage* theme is the
gap between **documented** architecture (Grafana, websockets/real-time,
CI/CD) and **shipped** architecture (none of those are wired in).

## Cross-cutting themes

1. **Tutorial-scaffold debt.** `frontend/.tutorial/` (29 tracked files, a whole
   separate Angular "housing" demo), `frontend/db.json` (json-server mock,
   frozen Jan-2025 data), identity leftovers (`package.json` name
   `angular.dev`, project `first-app`, component title `homes`), and dead
   `protractor`/`jasmine-marbles` deps all trace to an un-pruned tutorial.
2. **Docs ≠ reality.** README links to a deleted `ASSIGNMENT.md`, a missing
   `SOLUTION` (no extension), and a Discussions URL on an unrelated repo
   (`the-great-wall`); the repo name is spelled three ways
   (`sensor_app` / `sensor-data-app` / `the-great-wall`); SOLUTION.md
   advertises Grafana/websockets/CI/CD that don't exist in the repo.
3. **Not production-shaped.** Flask dev server + `debug=True`, wildcard CORS,
   data-generator thread spawned inside the app factory, no WSGI server, no
   `.dockerignore`, no committed lockfile, no healthchecks/restart policies,
   unpinned/EOL base images (Python 3.9 EOL, Postgres 13 EOL, `node:alpine`
   floating).
4. **No quality gates.** No CI, no pre-commit, no linters/formatters, no
   coverage, no security scanning — on either stack.

A reassuring counter-note: the **backend `app/` layering is genuinely clean**
— `database.py` exists solely to hold the shared `db` and break the import
cycle, the graph `routes → services → models/sensors → database` is acyclic,
and `instance/`, `.venv/`, `.idea/`, `.pytest_cache/`, `node_modules/` are all
correctly gitignored. This is competent work that simply stopped at MVP.

No issues were filed to GitHub — say the word and I'll open labelled tickets.

- Remove `frontend/.tutorial/`, `frontend/db.json`, dead deps, orphan `.pyc`.

claims (debug RCE, CORS wildcard, factory-spawned thread, `limit` coercion,

| #  | Area                          | Sev      | Finding (file:line) — fix |

| 6  | BackendError never wired      | Medium   | `errors.py` `BackendError` is never registered as a Flask error handler and is only raised inside the background thread — web error infra is effectively dead. |
| 7  | Tests un-importable from backend/ | Medium | No `backend/__init__.py`, no conftest/pytest.ini; tests do `from backend.app import …` (works only from repo root) while `run.py:3` uses `from app import …`. Two incompatible rootdirs. |

| 10 | Dead validation branch        | Low      | `routes.py:32` `not isinstance(limit, int)` can never be true — `get(type=int)` always yields int or the int default. |
| 11 | Timestamp serialized as HTTP-date | Medium | `services.py:50` emits raw `datetime` → Flask renders `"Fri, 26 Jun 2026 12:30:05 GMT"`, not ISO-8601 (lossy, naive value mislabeled GMT). Fix: `.isoformat()` + tz. |
